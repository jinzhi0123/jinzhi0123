1.Git安装

前往Git官网，点击Downloads下载。

选择对应的操作系统。

全部默认选择，Next即可。

安装完成后，在桌面/文件夹右键会出现两个Git的菜单按钮。

点击Git Bash Here进入Bash命令行。

输入git回车，得到如上提示则代表Git安装成功。

2.GitHub绑定

(1)创建GitHub仓库

首先，你需要一个GitHub账号，如果没有请先注册一个。

如果GitHub无法访问或速度过慢，可以尝试修改Hosts、使用CDN、科学上网等方法。

登录GitHub后，创建一个新的仓库

  • name设置为和用户名同名，这样开启pages首页后，访问地址不带二级目录。
  • 勾选 Add a README file 可以自动创建一个分支（现在GitHub默认主分支为main）

Gitee帮助文档：

如果你想你的 pages 首页访问地址不带二级目录，如ipvb.gitee.io，你需要建立一个与自己个性地址同名的仓库，如 https://gitee.com/ipvb 这个用户，想要创建一个自己的站点，但不想以子目录的方式访问，想以ipvb.gitee.io直接访问，那么他就可以创建一个名字为ipvb的仓库 https://gitee.com/ipvb/ipvb 部署完成后，就可以以 https://ipvb.gitee.io 进行访问了。

（三）利用GitHub Actions 将GitHub的仓库推送到Gitee

我们在前言说到，GitHub在国内的访问情况非常堪忧。那为什么不直接部署同样有Pages功能的Gitee呢？原因是Gitee Pages不会随着你的仓库更新而自动更新Pages页面，你需要手动去Gitee点击更新按钮（当然你可以选择购买Pro服务来获得自动更新功能）。

利用Actions，我们可以在仓库进行push、delete、 create等操作时自动执行一个脚本，将GitHub仓库自动推到Gitee仓库，并自动点击Pages更新按钮。

1.Gitee仓库建立

首先，你需要一个Gitee账号，如果没有，请前往Gitee官网注册。

登录账号后，创建一个与用户名同名的仓库，像GitHub一样：

2.密钥配置

(1)设置Secrets

在 GitHub 项目的「Settings -> Secrets」路径下配置好命名为 GITEE_RSA_PRIVATE_KEY 和 GITEE_PASSWORD 的两个密钥。其中：GITEE_RSA_PRIVATE_KEY 存放 id_rsa 私钥；GITEE_PASSWORD 存放 Gitee 帐号的密码。

(2)配置SSH密钥

GitHub密钥

前面我们我们在绑定GitHub时已经设置过SSH密钥了，这里不用再操作。

Gitee密钥

在 Gitee 的个人设置页面「安全设置 -> SSH 公钥」 配置 SSH 公钥（即：id_rsa.pub），命名随意。
